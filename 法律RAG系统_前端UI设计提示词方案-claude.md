# 民事诉状生成与法律咨询系统 —— 前端UI设计提示词方案

> 本方案用于指导设计师/AI设计工具（如 Claude、v0、Figma AI 等）产出"高端大气、简约克制、专业可信"的前端界面。可整体使用，也可按页面拆分单独使用。

---

## 一、总纲提示词（Master Prompt）

将以下内容作为系统级设计指令，放在每次生成页面之前：

```
你是一名资深法律科技（LegalTech）产品的高级UI/UX设计师，需要为一套
"民事纠纷诉状生成与法律咨询RAG系统"设计前端界面，技术栈为 Vue 3。

【产品定位】
面向普通用户与执业律师，帮助用户填写案件信息、上传证据、检索法律知识、
生成起诉状。用户对"专业性"和"可信度"极为敏感，界面必须传递出
"严谨、克制、可靠"的气质，而不是互联网消费类产品的活泼感。

【视觉基调】
- 主色：深蓝（如 #0F2A4A / #14315C），象征专业与信赖，用于导航、
  强调按钮、重要状态标识。
- 辅助色：石墨灰系（#4A5568、#718096、#A0AEC0）用于正文、边框、次要信息。
- 背景：近白灰（#F7F8FA）而非纯白，降低刺眼感，衬托内容卡片。
- 强调/状态色：仅在必要处使用，如成功（低饱和绿 #2F855A）、
  警示（低饱和橙 #C05621）、错误（低饱和红 #C53030）。禁止使用
  高饱和度、渐变炫彩、卡通风格插画。
- 禁止：大面积渐变、圆润卡通图标、跳动动效、鲜艳撞色、娱乐化文案。

【版式与间距】
- 采用 8px 网格系统，页面留白充分，信息密度适中偏低，避免拥挤。
- 内容区域最大宽度建议 1200～1280px，居中布局，两侧留白。
- 卡片圆角统一使用小圆角（4～8px），体现克制而非圆润可爱风格。
- 阴影使用极浅的投影（如 0 1px 3px rgba(0,0,0,0.06)），不做强立体感。

【字体】
- 中文正文：思源黑体 / PingFang SC / Microsoft YaHei 等无衬线字体。
- 标题字重适度加粗（500～600），正文常规（400），避免过粗造成压迫感。
- 字号层级建议：H1 24px / H2 20px / H3 16px / 正文 14px / 辅助文字 12px。
- 行高不小于 1.5，保证长段落法律文本的可读性。

【组件与交互】
- 表单类组件（输入框、下拉、步骤条）风格统一、边框细腻（1px 实线），
  避免夸张的悬浮特效；聚焦态用主色细线框，不使用大面积高亮色块。
- 按钮分级明确：主操作（深蓝实心）、次操作（描边或灰底）、
  危险操作（低饱和红），禁止一屏出现过多同权重按钮。
- 全局使用统一的空状态、加载状态、错误状态设计，风格克制、
  文案专业（不用卡通插画、不用俏皮文案）。
- 所有 AI 生成内容、免责声明需要有统一的"提示条"组件样式
  （浅灰底 + 图标 + 小字），在关键页面固定展现，不可被忽略但不过分打扰。

【合规与信任元素（必须体现）】
- 首页、法律咨询回答末尾、起诉状生成/预览页需醒目但不刺眼地展示：
  "以上内容仅供参考，不构成正式法律意见，具体案件建议咨询执业律师"
  "AI生成内容仅供参考，请核实后使用"
- 每一次AI回答、每一份生成文书都需可追溯（标注引用法条/知识库来源）。

请基于以上设计语言，输出该系统各页面的具体设计方案（可为设计稿描述、
Figma 组件规范，或 Vue3 + 组件库的实现方案）。
```

## 二、通用交互与信息架构提示词

```
请为系统设计统一的导航与信息架构：
1. 左侧固定导航栏（深蓝底色/浅灰底色可二选一，与主色呼应），
   一级菜单：法律咨询、案件信息、证据材料、起诉状生成、知识库管理、
   （管理员视角下）系统设置。
2. 顶部栏放置：当前用户信息、消息/通知入口、退出登录。
3. 面包屑或步骤条在多步骤页面（案件信息填写、起诉状生成）中固定展示，
   让用户随时知道自己所处流程节点。
4. 所有列表页统一采用表格/卡片 + 筛选栏 + 分页的组合，操作列采用
   图标+文字或纯图标（配 tooltip），保持视觉一致性。
5. 全局提供"AI生成内容仅供参考"的通用页脚或角标组件，可在需要的页面复用。
```

## 三、分页面设计提示词

### 1. 登录页

```
设计一个简洁、可信赖的登录页：
- 左右分栏布局：左侧为品牌视觉区（深蓝背景 + 简洁的法律元素线性图形，
  如天平、法典轮廓，线条化处理，不使用卡通插画），右侧为登录表单。
- 表单包含：账号、密码、图形验证码（3个字段），
  验证码采用简洁的干扰线+扭曲文字样式，旁边配"看不清，换一张"链接。
- 登录按钮使用主色实心大按钮，下方提供"没有账号？去注册"的次要链接。
- 页面底部小字展示系统名称与合规声明（如"AI生成内容仅供参考"提示的
  简短版本 + 版权信息）。
- 整体避免大面积留白空洞感，也避免元素堆砌，视觉重心居中偏上。
```

### 2. 注册页

```
设计与登录页视觉统一的注册页：
- 同样左右分栏或与登录页共用布局框架，右侧表单包含4个字段：
  账号、密码、确认密码、图形验证码。
- 密码强度提示（弱/中/强）以细窄进度条+文字呈现，颜色克制
  （灰→橙→绿，不用鲜艳色）。
- 两次密码不一致时，在确认密码框下方即时给出细字号红色错误提示，
  不使用弹窗打断。
- 注册按钮下方提供"已有账号？去登录"链接。
```

### 3. 法律咨询对话页

```
设计一个类ChatGPT但更"专业严肃"的法律咨询对话界面：
- 三栏布局：
  左栏：会话列表（支持新建、重命名、删除会话，会话名可编辑，
        当前选中会话高亮为浅蓝底）。
  中栏：对话主区，用户消息右对齐（浅灰气泡），AI回答左对齐
        （白底卡片 + 细边框，非气泡感，更接近"文档"质感），
        AI回答中引用的法条/来源以可点击的角标或行内标签呈现
        （如 [《民法典》第577条]），点击可展开来源详情。
  右栏（可折叠）：模型与参数设置面板，包含：
        知识库多选（法律法规库/司法解释库/类案案例库，多选框形式）、
        模型选择下拉、温度/topP滑杆（带数值显示）、
        最长输出token数输入框、历史对话轮数选择。
- 每条AI回答末尾固定展示浅灰底提示条："以上内容仅供参考，
  不构成正式法律意见，具体案件建议咨询执业律师"。
- 输入框下方提供常见咨询类型快捷标签（诉讼时效/管辖法院/
  举证责任/诉讼费用/诉讼流程），点击自动填充问题模板。
- 支持流式输出的打字机效果展示，输出过程中显示细线条加载指示，
  不用夸张的跳动动画。
```

### 4. 案件信息填写页

```
设计一个分步骤（Step表单）的案件信息填写页面：
- 顶部横向步骤条，包含：当事人信息 → 案由与诉讼请求 →
  事实与理由 → 证据清单 → 确认提交，共5步，当前步骤高亮为主色。
- 每步表单采用两栏或单栏卡片布局，字段分组清晰（原告信息/被告信息
  分组展示，各自可增加多个当事人）。
- 案由字段为下拉+可搜索选择（合同纠纷/侵权纠纷/婚姻家庭纠纷/
  劳动争议等），选择后在表单下方以浅蓝提示卡片展示
  "根据该案由，可能适用的法律条文"推荐列表，可折叠查看。
- 每步右下角提供"保存草稿"（次要按钮）与"下一步"（主按钮），
  顶部或侧边显示"草稿已自动保存于 xx:xx"的细字提示，
  降低用户对丢失数据的焦虑。
- 最后一步提供信息总览，用只读卡片形式展示前面所有已填内容，
  支持点击跳回对应步骤修改。
```

### 5. 证据材料上传页

```
设计证据材料上传与管理页面：
- 顶部为拖拽/点击上传区域，支持图片、PDF、聊天记录截图、
  录音转文字稿，上传区域使用虚线边框+简洁上传图标，
  不使用花哨的动效。
- 上传列表以卡片网格或表格形式展示每份材料的缩略图/文件图标、
  文件名、上传时间、OCR识别状态（识别中/已完成/失败，
  用小圆点+文字状态标签，颜色低饱和）。
- 每份材料支持分类标注（书证/物证/视听资料等），
  以下拉选择或标签选择方式呈现在卡片上。
- 点击材料可展开侧边抽屉，展示OCR识别出的关键信息
  （合同金额、签署日期、当事人姓名等），以结构化表单形式呈现，
  支持人工核对与修正，修正处需有醒目但克制的"待确认"标记。
```

### 6. 起诉状生成与预览页

```
设计起诉状生成与预览页面，体现"正式法律文书"的严肃质感：
- 左侧为生成过程/引用溯源面板：展示生成所依据的案件信息摘要、
  引用的法条、知识库来源、匹配的文书模板类型，
  以可折叠的信息卡片呈现，方便用户核对依据。
- 右侧主区域为文书预览区，采用接近A4纸张比例的白色文档卡片，
  仿照正式起诉状排版（标题居中、当事人信息、诉讼请求、
  事实与理由、证据清单、致送法院、落款），字体使用宋体/衬线体
  以贴近正式文书视觉习惯，与页面其余部分的无衬线UI字体形成
  "系统界面 vs 文书正文"的有意区分。
- 文档区域支持富文本在线编辑（选中即可修改字号加粗等基础排版），
  顶部工具栏提供"导出Word""导出PDF""重新生成"按钮。
- 页面顶部或文书上方固定展示醒目提示条（浅橙底或浅灰底 + 警示图标）：
  "AI生成内容仅供参考，请核实后使用"，不可被折叠隐藏。
```

### 7. 知识库解析页（离线文档导入）

```
设计知识库文档解析配置页面：
- 上传区支持PDF、Word、Markdown、图片等类型，
  与证据上传页保持一致的上传交互，但视觉上归属于"知识库管理"
  模块的配色分区（可用同一主色系但通过侧边导航状态区分场景）。
- 上传完成后展示"解析参数配置"表单：文本切块大小（滑杆+数值输入）、
  重叠大小（滑杆+数值输入）、检索类型（稠密向量检索/混合检索，
  单选卡片形式，附简短说明文字解释二者区别）。
- 提供"预览切块效果"按钮，点击后以列表形式展示切分后的前几个
  文本块，帮助用户直观判断参数是否合适。
- 底部提供"开始解析"主按钮，解析过程以进度条+状态文字展示
  （排队中/解析中/向量化中/已完成）。
```

### 8. 知识库预览页

```
设计知识库文档块预览与编辑页面：
- 左侧为该文档的分块列表（简洁列表，每项展示块序号+前几十字摘要），
  右侧为选中块的详细内容展示区，支持直接编辑文本内容并保存，
  支持删除该文档块（删除需二次确认，使用轻量确认弹窗而非页面跳转）。
- 顶部展示该文档的基本信息（文件名、所属知识库、切块参数、
  总块数），使用面包屑或信息条形式，风格克制不抢主体内容视觉焦点。
```

### 9. 知识库管理页

```
设计知识库管理主页：
- 顶部为知识库卡片列表（如"法律法规库""司法解释库""类案案例库"），
  每张卡片展示知识库名称、文档数量、最近更新时间、简短描述，
  卡片风格与其余模块一致（细边框、小圆角、浅阴影），
  提供"新建知识库"入口（虚线卡片+加号，风格克制不浮夸）。
- 点击进入某知识库后，展示该库下的文档列表（表格形式：
  文件名、类型、大小、解析状态、上传时间、操作），
  点击"预览"进入"知识库预览页"，操作列提供重新解析/删除等。
- 页面顶部提供"上传文档"按钮，跳转或弹出至"知识库解析页"流程。
```

## 四、组件规范建议提示词（可用于生成设计系统/组件库）

```
请基于上述视觉基调，输出一套可复用的 Vue3 组件设计规范，至少包含：
1. Design Tokens：色板（含语义化命名，如 --color-primary、
   --color-bg-page、--color-text-secondary 等）、字号/字重/行高、
   间距（4/8/12/16/24/32px 阶梯）、圆角、阴影。
2. 基础组件：Button（primary/secondary/danger/text 四类）、Input、
   Select、Stepper、Tag、Badge（状态标识）、Card、Table、Drawer、
   Modal（含二次确认样式）、Upload、Alert/Notice（用于合规提示条）。
3. 业务组件：ChatMessageBubble（AI/用户两种样式）、
   CitationTag（法条引用角标）、DocumentPreviewCard（文书预览卡）、
   StepFormLayout（分步表单外层容器）。
每个组件需说明使用场景、状态（默认/悬浮/聚焦/禁用/加载/错误）
及对应视觉规范，保持整体气质"专业、克制、可信赖"。
```

## 五、使用建议

1. **整体一次性生成**：可将"一、总纲提示词"+"二、通用交互"+"三、全部分页面提示词"依次拼接，交给设计/前端生成工具，一次性产出完整设计方案。
2. **单页迭代**：也可只取"三"中的某一小节单独喂给工具，针对某个页面反复打磨。
3. **配合 Claude Code / 前端设计能力**：将本方案作为系统提示词的一部分，配合已梳理的页面清单（登录、注册、法律咨询对话、案件信息填写、证据上传、起诉状生成预览、知识库解析/预览/管理共9个页面），逐页生成 Vue 组件代码。
4. 建议先用"总纲提示词"生成设计系统（配色、字体、间距、组件），再逐页生成，保证全站视觉统一。
